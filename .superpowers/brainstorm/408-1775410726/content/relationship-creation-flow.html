How does the user create a relationship with row-level handles?

Two approaches for the creation flow. They can also be combined. Click to select.

A

Drag from row handle (replaces right-click flow)

User drags from a source attribute handle → drops on a target table or row → modal opens with type/color/cardinality. Attributes auto-filled from drag source/target.

✓ Pros
Very direct — source + target attributes picked by gesture.
No extra modal fields needed.
Phase 6 FD drag uses exactly the same handle pattern.
✗ Cons
Right-click context menu "Add relationship" becomes redundant or needs repurposing.
Relationship type → still needs a modal after drop.
B

Right-click → modal gains "from / to attribute" selectors

Keep the existing right-click flow from Phase 13. RelationshipEditModal gets two new dropdowns: source attribute and target attribute. Handles on rows are display-only (show where the edge exits).

C

Both — drag OR right-click (recommended)

Drag from a row handle = fast path, attributes auto-selected. Right-click header = still works, modal includes attribute selectors as a fallback/edit path. Same handles serve both FDs (Phase 6) and relationships.

How it works together:
  • Hovering a row reveals the handle dot on the right side
  • Drag from handle → drop on target table (any area) → modal opens, source attribute pre-filled, user picks target attribute + type/cardinality
  • Drag from handle → drop on a specific target row → both source + target pre-filled, modal just confirms type/cardinality
  • Right-click header → "Add relationship" still works → modal opens with all fields including attribute selectors (nothing pre-filled)
  • Phase 6 FD drag uses left-side handles on the same rows — no conflict
Note on data model: All options require adding attr1Id and attr2Id to the Relationship store shape (currently only has table1Id / table2Id). Both can be optional/null for table-level relationships if needed.
